# ADR-0001 — Modular foundation for Self-Evolving System

## Статус
Accepted

## Контекст

Проект формируется как система, которая со временем должна улучшать собственную способность:
- разрабатывать агентов;
- разрабатывать скиллы;
- накапливать качественные паттерны;
- улучшать отдельные части себя на основе исследований и проверки эффекта.

На раннем этапе возник риск смешать в одну папку:
- разработку скиллов;
- разработку агентов;
- исследования;
- правила системы;
- память;
- артефакты.

Это делало бы систему трудной для расширения и анализа.

## Решение

Принято строить систему модульно.

Базовые модули:
- `core`
- `modules/research-loop`
- `agents-lab`
- `skills-lab`
- `memory`
- `evaluations`
- `artifacts`

Отдельный `prompts-lab` пока не вводится, так как соответствующие задачи временно закрываются через `agents-lab` и `skills-lab`.

## Последствия

### Плюсы
- проще улучшать модули по отдельности;
- понятнее, где искать правила, где исследования, где результаты;
- легче запускать первый MVP;
- меньше смешения сущностей.

### Минусы
- придётся поддерживать дисциплину границ между модулями;
- часть модулей на старте будет минимальной;
- возможна последующая реорганизация, когда появятся новые реальные потребности.

## Следствие для первого этапа

Первым рабочим контуром становится `modules/research-loop`, который должен дать системе первый повторяемый цикл самоулучшения.
