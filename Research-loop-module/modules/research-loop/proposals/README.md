# Proposals

Proposal — это мост между исследованием и внедрением.

Proposal должен быть:
- конкретным;
- маленьким;
- проверяемым;
- адресным.

Плохой proposal:
- "улучшить систему"
- "усилить агента"

Хороший proposal:
- "добавить в agent instructions обязательный раздел 'tool selection constraints'"
- "ввести короткий self-check перед финальным ответом"
