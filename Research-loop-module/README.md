# Self-Evolving System

Self-Evolving System — это модульная система, цель которой со временем улучшать собственную способность создавать более сильных агентов, более качественные скиллы и более устойчивые рабочие паттерны.

На текущем этапе система строится как **hybrid architecture**:
- горизонтальный канонический слой правил и памяти;
- вертикальный операционный слой run/campaign для одного завершённого цикла.

## Основная идея

Система развивается через цикл:

1. исследование качественных источников;
2. извлечение полезных паттернов;
3. формулирование предложений;
4. внедрение небольших изменений;
5. оценка эффекта;
6. накопление удачных решений в памяти.

## Что входит в систему

- `core/` — базовые правила, архитектура, ADR и контракты
- `modules/research-loop/` — первый MVP-контур самоулучшения
- `agents-lab/` — среда разработки агентов
- `skills-lab/` — среда разработки скиллов
- `memory/` — принятые и проверенные паттерны
- `evaluations/` — чеклисты и критерии оценки
- `artifacts/` — общие производные материалы

## Архитектурный режим (гибрид)

### Горизонтально (canonical)

- `memory/`
- `evaluations/`
- `agents-lab/`
- `skills-lab/`
- общие policy/contracts/runbook в `core/`

### Вертикально (operational trace)

- `modules/research-loop/campaigns/<date-topic>/`
- в campaign лежит полный комплект: `plan.md`, `sources.md`, `reviews.md`, `proposal.md`, `experiment.md`, `decision.md`, `artifacts/`

## Что не выделяется отдельно на этом этапе

Сейчас нет отдельного `prompts-lab`, потому что его функции временно покрываются `agents-lab` и `skills-lab`.
