# System Blueprint

## Назначение

Self-Evolving System — это проект, который должен постепенно повышать качество собственных результатов через модульное улучшение.

Система улучшает не всё сразу, а отдельные блоки:
- подходы к разработке агентов;
- подходы к разработке скиллов;
- внутренние правила и шаблоны;
- накопление проверенных паттернов;
- исследовательские процессы.

## Архитектурный принцип

Каждый модуль должен быть улучшабелен отдельно.

Это означает:
- у каждого модуля должна быть понятная роль;
- улучшения должны приходить в модуль адресно;
- эффект изменений должен быть наблюдаем;
- удачные паттерны должны накапливаться;
- неудачные попытки тоже должны фиксироваться, чтобы не повторять их бесконечно.

## Базовые модули

### 1. Core
Содержит описание системы, правила и архитектурные решения.

### 2. Research Loop
Собирает, проверяет и превращает знания во внедряемые предложения.

### 3. Agents Lab
Место, где развиваются подходы к созданию агентов.

### 4. Skills Lab
Место, где развиваются подходы к созданию скиллов.

### 5. Memory
Хранилище принятых и проверенных паттернов.

### 6. Evaluations
Критерии и способы проверки, стало ли лучше.

### 7. Artifacts
Производные артефакты: отчёты, сборки, временные файлы.

## Первый приоритет

Первый приоритет — не расширение количества модулей, а запуск простого воспроизводимого `research-loop`, который:
1. берёт тему;
2. собирает качественные источники;
3. фильтрует их;
4. формулирует предложения;
5. проверяет их;
6. записывает результат.

## Ограничение качества

Желательно опираться в первую очередь на проверенные источники:
- OpenAI
- Anthropic
- Google
- хорошие технические репозитории и качественный проверенный код

Если источник слабый, это должно быть явно отмечено.
