# MVP Change Checklist

Используй этот чеклист для маленьких изменений.

## Перед внедрением
- [ ] изменение сформулировано конкретно
- [ ] понятно, какой модуль меняется
- [ ] понятен ожидаемый эффект
- [ ] изменение достаточно маленькое для пробного теста

## Во время проверки
- [ ] есть понятный кейс для теста
- [ ] зафиксирован Baseline (базовая линия / поведение ДО изменения)
- [ ] сравнивается не абстрактное ощущение, а наблюдаемое поведение (До и После)
- [ ] проверено, что старое работающее поведение (инварианты) НЕ сломалось (отсутствие регрессий)
- [ ] фиксируются неожиданные побочные эффекты

## После проверки
- [ ] можно честно сказать: better / worse / unclear на основе объективного сравнения с Baseline
- [ ] результат записан
- [ ] если улучшение подтверждено, оно перенесено в `accepted-patterns.md`, и связанный базовый шаблон ОБНОВЛЕН
- [ ] если улучшение не подтверждено, причина отказа записана, паттерн перенесен в `anti-patterns.md` (чтобы избежать повторения)
