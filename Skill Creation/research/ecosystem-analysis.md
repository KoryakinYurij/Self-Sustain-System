# Исследование существующих экосистем Agent Skills и AI Rules (Status: SOTA 2026)

На основе аудита современных практик (март 2026) экосистема ИИ-правил пережила тектонический сдвиг. Фокус сместился с монолитных community-driven файлов `.cursorrules` на **стандартизированные протоколы**, **модульные "скиллы"** и **официальные инициативы**, такие как Agentic AI Foundation (AAIF) под эгидой Linux Foundation.

Ниже представлен анализ надежных SOTA-подходов и устаревающих практик (legacy), на которые стоит опираться при построении нашей продвинутой архитектуры Agent Skills.

## 1. Фундамент и официальные стандарты (High Reliability)

### [Agentic AI Foundation (AAIF) & AGENTS.md](https://agents.md)
В конце 2025 года OpenAI, Anthropic и другие передали стандарт `AGENTS.md` и `MCP` в Linux Foundation (AAIF). 
- **AGENTS.md**: Кросс-платформенный "минимальный общий знаменатель". Работает как `README` для агентов (поддерживается Cursor, GitHub Copilot, Cline, Windsurf).
- Это **самый надежный фундамент** для корневой конфигурации любого сложного проекта.

### [anthropics/skills](https://github.com/anthropics/skills)
Официальный репозиторий Anthropic, задающий архитектурный SOTA-паттерн ("прогрессивное раскрытие").
- **Модель**: `skill = папка + SKILL.md` (YAML frontmatter + Markdown) + `scripts/` + `resources/`.
- Агент читает только метаданные, а полные инструкции и тяжелые ресурсы подгружает динамически, чтобы не перегружать контекст. Это идеальный шаблон для нашей системы.

### Экосистема OpenAI (Skills + Shell + Compaction)
Критическая практика для долгоживущих агентов:
- **Skills**: Версионированные наборы инструкций.
- **Shell**: Песочница/терминал, где агент запускает код (MCP, Gemini CLI) вместо того, чтобы просто писать текст.
- **Compaction**: Сжатие контекста сервером для сложных workflow (например, многоэтапный деплой инфраструктуры).

- **Антипаттерн:** В 2026 году текстовые guardrails считаются слабыми. Реальная безопасность закрывается технологически: песочницы (Shell), пре-коммиты, policy-as-code и Evals (оценка выполнения), а не просто словами "не делай это".

## 2. Источники для крафта скиллов (Skill-Authoring Focus)

> [!IMPORTANT]
> Мы **не строим агентов и фреймворки**. Мы пишем **скиллы**, которые агенты потребляют. Всё ниже отобрано по одному критерию: **помогает ли это написать лучший SKILL.md, скрипт или тест?**

### Первоисточники (читать и перечитывать)

#### [Anthropic — The Complete Guide to Building Skills for Claude](https://docs.anthropic.com/en/docs/agents-and-tools/skills)
**Самый важный документ для нас.** Покрывает полный цикл: планирование → структура → тестирование → распространение. Ключевые выводы:
- **Micro-skills > монолиты**: маленькие фокусные скиллы, которые можно цепочкой, надёжнее одного большого.
- **Hand-edit всегда**: даже если Claude сгенерировал скилл — обязательно редактируй руками.
- **3 уровня тестирования**: manual (Claude.ai) → scripted (Claude Code) → programmatic (API).
- **Метрики**: trigger rate, tool calls, token consumption — что измерять и как.

#### [agentskills.io — Open Standard Specification](https://agentskills.io)
Формализованная спецификация формата скиллов, принятая OpenAI Codex, Claude Code, Cursor, VS Code, GitHub Copilot.
- **Валидация**: CLI-валидатор `skills-ref validate` для проверки SKILL.md.
- **Правила**: `name` должен совпадать с именем директории, `description` 1-1024 символов с ключевыми словами для trigger.
- **Рекомендации по body**: step-by-step guides, input/output примеры, edge cases.

#### [anthropics/skills — GitHub Repository](https://github.com/anthropics/skills)
Живые reference-реализации скиллов. Что конкретно изучать:
- **`skills/`** — production-ready примеры (Excel, PowerPoint, document creation, slack-gif-creator).
- **`tests/`** — как Anthropic сами тестируют свои скиллы.
- **Паттерны `scripts/`** — Python-скрипты для детерминированных операций (PII-редакция, сложные трансформации данных) вместо надежды на LLM.

### Обучение (курс)

#### [DeepLearning.AI — Agent Skills with Anthropic](https://www.deeplearning.ai/short-courses/agent-skills-with-anthropic/)
Короткий курс от DeepLearning.AI + Anthropic (Elie Schoppik). Практический, не теоретический:
- Структура skill-папок и прогрессивное раскрытие.
- Skills vs Tools vs MCP vs Subagents — когда что использовать.
- Создание custom-скиллов + комбинирование с MCP.
- Демо на Claude.ai, Claude Code, API, Agent SDK.

### Тестирование и качество

#### [Anthropic — Evals & Testing](https://docs.anthropic.com/en/docs/build-with-claude/develop-tests)
Как тестировать поведение агента (и скиллов в частности):
- **Code-based graders**: для детерминированных проверок (скрипт вернул ожидаемый JSON).
- **LLM-as-a-judge**: модель оценивает качество open-ended выхода скилла.
- **CI/CD integration**: автоматические evals при каждом изменении скилла.

> [!TIP]
> **Практическое правило**: если скилл содержит критическую операцию (деплой, удаление, запись в БД) — оборачивай в Python-скрипт из `scripts/`, а не полагайся на LLM. Скрипт детерминирован, быстр, тестируем.

## План действий: Что изучать и внедрять (Must Study)

Для построения надежной SOTA-системы Agent Skills, мы **исключаем** изучение старых монолитных `.cursorrules` и фокусируемся на следующем:

### 1. Внедрение AGENTS.md как Корневого Стандарта
- **Цель:** Заменить разрозненные `.cursorrules` или `.clinerules` на единый `AGENTS.md`.
- **Что изучать:** Структуру `AGENTS.md` от OpenAI/AAIF. Выявить, какие базовые конвенции (навигация, sysadmin-права, формат ответов) должны жить в корне, а какие — уходить в скиллы.

### 2. Принятие Архитектуры Прогрессивного Раскрытия (`anthropics/skills`)
- **Цель:** Перестроить архитектуру навыков по децентрализованной модели Anthropic.
- **Что изучать:** Как эффективно структурировать `SKILL.md` + `scripts/` + `resources/`. Как добиться того, чтобы агент загружал тяжелые мануалы по Vector DBs только тогда, когда пользователь запрашивает интеграцию с ИИ, экономя токены и внимание агента.

### 3. Shell-автоматизация, MCP и Executable Skills (OpenAI Pattern)
- **Цель:** Переход от текстовых инструкций к исполняемому рабочему процессу.
- **Что изучать:** 
  - Как интегрировать Model Context Protocol (MCP) для безопасного управления Docker и сервером.
  - Написание скриптов автоматизации (Gemini CLI), вызываемых агентом напрямую из скилла, вместо того, чтобы просить пользователя "скопировать и вставить эту команду".

### 4. Policy-as-Code и Защитные Evals (Guardrails)
- **Цель:** Истинная безопасность и качество без галлюцинаций.
- **Что изучать:** Отказ от текстовых запретов ("не ломай продакшен"). Внедрение тестирования поведения (Evals) и скриптов проверки (например, `test-docker-compose.sh`), которые агент обязан запускать и парсить в JSON Schema (Structured Outputs) перед выдачей финального ответа.
